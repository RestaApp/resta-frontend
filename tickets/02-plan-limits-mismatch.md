# Тикет 2 — Завышенные месячные лимиты планов в окружении

**Тип:** config / data
**Приоритет:** средний

## Проблема
`GET /api/v1/subscriptions/current` под рестораном отдаёт `restaurant_free.limits`:
```json
{ "monthly_vacancies": { "used": 2, "limit": 30, "remaining": 28 },
  "monthly_replacements": { "used": 1, "limit": 10, "remaining": 9 } }
```
То есть **30 вакансий / 10 замен**, тогда как по MONETIZATION.md должно быть **3 / 1**.

## Последствие
Лимит почти не исчерпывается → бэк не возвращает **402** → 402-флоу покупок слотов невозможно протестировать вживую (ни покупку, ни индикатор «лимит исчерпан»).

## Что сделать
Привести лимиты планов к спецификации:
- `restaurant_free`: 3 вакансии + 1 замена / мес
- `employee_free`: 1 замена / мес

**Либо** явно подтвердить, что лимиты намеренно подняты для тест-окружения (тогда закрыть как «by design»).

## Критерий готовности
Лимиты планов соответствуют MONETIZATION.md, либо зафиксировано осознанное отличие тест-окружения.

## Контекст
Фронт читает `limits`/`usage` как есть и показывает индикатор «осталось N из M» — правок не требует. Связан с [Тикетом 1](./01-purchase-prices-not-configured.md): без 402 и без цен покупка слота не доходит до оплаты.
